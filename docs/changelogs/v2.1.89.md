## unlock-commands

将 `commit`、`commitPushPr`、`initVerifiers`、`version` 从内部命令列表移至公开命令列表，移除 `/version` 的 ant-only 限制，所有用户均可使用这些命令。

## fix-swarm-banner

修复 swarm banner 边框未考虑 CompanionSprite 宽度的问题。当 `autoNameSessionFromPlan` 在 plan→auto 转换时设置 `standaloneAgentContext.name`，banner 激活后其边框超出可用宽度导致换行，破坏 prompt 布局。

## buddy

运行 `/buddy hatch` 进入交互式创建流程，依次选择物种、稀有度、闪光、眼睛、帽子、名字、性格，然后可以描述你想象中同伴的样子（或跳过），最后自动生成中文角色 profile。

### 怎么使用宠物系统

- 运行 `/buddy`：查看当前正在展示的宠物卡片
- 运行 `/buddy hatch`：领养一只新宠物
- 运行 `/buddy list`：查看所有已领养宠物，并在列表中直接切换当前展示宠物
- 运行 `/buddy card`：查看当前宠物卡片（精灵图、种族、属性、profile）
- 运行 `/buddy chat <消息>`：直接和当前宠物聊天，回复会显示在气泡中
- 运行 `/buddy pet`：摸一摸当前宠物
- 运行 `/buddy mute` / `/buddy unmute`：隐藏 / 展示当前宠物系统

### 宠物系统规则

- 可以同时领养多只宠物，但同一时刻只展示一只
- 每只宠物的属性、profile 都独立保存
- 在 `/buddy list` 中切换展示宠物时，不会丢失原宠物的成长进度
- 每次和 AI 对话后，当前宠物会在气泡中发表结合上下文的短评（基于 profile、物种、性格）
- 气泡框加宽（wrap 40 字符，box 44），支持更长的上下文相关评论
- 物种基础属性（base stats）按 T1–T4 梯队分档
- 每轮对话自动获得经验值（等于本轮消耗的 input + output tokens），闪光宠物 ×1.2
- 升级公式：`累计经验 = 3000 × 等级²`，满级 100 约需 3000 万 tokens（重度用户约一个月）
- 每轮对话有 10% 概率触发努力值（EV）成长，由 Haiku 判断对话偏向哪个维度后加点
- 努力值上限：单项 252，总计 510；使用AI时有概率让宠物努力值 +1，闪光宠物每次 +2
- 性格自带性格修正：8 种性格各有一项属性 +10%、一项 -10%
- 属性实时计算，公式：`floor(((2×种族值 + 个体值 + 努力值/4) × 等级 / 100) + 5) × 性格修正)`
- 孵化时可自由调整个体值（0–31），进度条式交互
- 宠物卡片显示等级、性格修正、经验进度条和实时计算的五维属性
- 运行 `/buddy release`：放生一只宠物（不可撤销）
- 宠物数据独立存储在 `~/.claude/buddy.settings.json`，不污染 settings.json
- 用户在创建时输入的想象描述，会作为 profile 生成的最高权重输入
- 生成加载界面使用旋转动画（◐◓◑◒）

## token-budget

开启 Token 预算功能，允许用户在消息中指定 token 消耗目标，Claude 会持续工作直到接近预算上限，适合需要深度思考或大量编码的复杂任务。

用法：
- 在消息开头或结尾加 `+500k`、`+2m` 等简写形式
- 或使用完整形式：`use 500k tokens`、`spend 2m tokens`
- k = `千`，m = `百万`，b = `十亿`

## terminal-capture

开启终端面板捕获功能，允许 Claude 读取内置终端面板（Alt+J）中的内容。

功能：
- 新增 `terminal_capture` 工具，可捕获终端面板中当前显示的文本内容
- 支持 `lines` 参数控制捕获的滚动历史行数（1–10000）
- 通过 tmux 会话实现持久化终端面板的内容读取
- 工具为只读且并发安全，已加入 YOLO 模式安全白名单

## 彩虹色

- `/`开头的指令（例如`command`、`skill`）变为彩虹色

## mcp-skills

开启 MCP Skills 发现功能，允许 Claude Code 从 MCP 服务器自动发现和加载 `skill://` 资源作为可用 skills。

功能：
- MCP 服务器可通过 `skill://` URI 前缀暴露 skill 资源
- Claude Code 连接 MCP 服务器时自动发现并加载这些 skills
- 支持标准 skill frontmatter（name、description、when-to-use 等）
- MCP skills 出现在可用 skills 列表中，可被模型自动调用
- 安全限制：MCP skills 中的 inline shell 命令不会被执行

## compaction-reminders

开启压缩提醒功能，在上下文压缩后向模型附加提醒信息，帮助 Claude 在压缩后保持对关键上下文的感知。

## remove-anti-distillation-restriction

移除反蒸馏限制，避免正常开发、研究和使用路径被额外防护误伤。

功能：
- 不再向请求体注入 `anti_distillation` / `fake_tools` 相关参数
- 移除 `ANTI_DISTILLATION_CC` 相关公开说明
- 清理相关反蒸馏表述，同时保留独立的 `CONNECTOR_TEXT` 与 `STREAMLINED_OUTPUT` 功能

## remove-npm-to-native-install-notice

移除 “Claude Code has switched from npm to native install...” 提示，减少无关安装迁移信息对正常使用的打扰。

功能：
- 不再展示 npm 切换到 native install 的提示信息
- 保持其他通知逻辑不变

## remove-unauthorized-text-modifications

移除 CC CLI 擅自对用户文本的编辑操作，确保工具不会在未经用户同意的情况下修改提交信息或 PR 内容。

变更：
- 移除 git commit 中自动追加的 `Co-Authored-By: Claude ... <noreply@anthropic.com>` 尾部标记
- 移除 PR 描述中自动追加的 `🤖 Generated with [Claude Code](...)` 归属文本
- 移除带统计信息的增强版 PR 归属（贡献百分比、N-shotted、模型名称等）
- 清理 BashTool 系统提示、`/commit`、`/commit-push-pr` 命令中的归属模板
- 移除 settings 中已无效的 `attribution.commit`、`attribution.pr`、`includeCoAuthoredBy` 配置项
- 清理 updateConfig skill 中的归属配置文档

## skip-login-third-party-api

配置了第三方 API 的用户首次启动时不再弹出 OAuth 登录流程。当同时设置了 `ANTHROPIC_API_KEY` 和指向非 Anthropic 官方地址的 `ANTHROPIC_BASE_URL` 时，系统自动识别为第三方 API 用户，跳过登录步骤直接进入使用。主题选择、安全提示等其他引导步骤保持不变。

## add-bypass-permissions-mode

将 bypass permissions 模式加入 shift+tab 权限模式循环，成为第四个可切换模式。用户无需通过 CLI 参数启动即可在会话中随时开启该模式，跳过所有权限确认弹窗。

模式循环顺序：default → accept edits → plan mode → bypass permissions → default。

## remove-commit-attribution

移除 COMMIT_ATTRIBUTION 功能，彻底清理 Claude 对 git commit 贡献比例追踪的底层基础设施。

变更：
- 不再逐字符追踪 Claude 在每个文件中的贡献比例
- 不再在 commit 信息或 PR 描述中生成贡献统计（如"Claude contributed 73%"）
- 不再在 git prepare-commit-msg 钩子中自动注入归属信息
- 会话存储和恢复流程更轻量，不再携带归属快照数据
- 已有会话记录中的旧归属数据向后兼容，静默忽略
